#### 2.2 Verificação das Personas

<center>

**Tabela 4** - Checklist para a Verificação das Personas.

| ID | Item de Verificação | Autor(es) | Referência |
|------|-------|------|---------|

| 7   | A persona primária possui elasticidade? (_Elasticidade: é quando uma persona acaba por ter características muito generalizadas. Assim, uma persona elática pode induzir a equipe de desenvolvimento de projeto a seguir cainhos incorretos ou menos produtivos para a concepção do produto._<a id="FTF1" href="#FTF1Ref">^1^</a>  )  |  |  | 
| <a href="#REF02">2</a> | A persona primária é sólida e tangível o suficiente para colocar os pressupostos de design em perspectiva? (_Solidez/Tangibilidade: é o oposto de uma persona elástica, quando uma persona é considerada sólida é porque seu nível de especialização é bem definido para o software sendo analisado no momento._<a id="FTF1" href="#FTF1Ref">^1^</a>) |  |  |
